<h2 class="h2b" id="calibre_link-372"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-1094" class="calibre1"></span><strong class="calibre2"><span class="big">12</span><br class="calibre12" />WEB SCRAPING</strong></h2>
<div class="image"><img src="../images/000070.jpg" alt="Image" class="calibre3" /></div>
<p class="noindent">In those rare, terrifying moments when I’m without Wi-Fi, I realize just how much of what I do on the computer is really what I do on the internet. Out of sheer habit I’ll find myself trying to check email, read friends’ Twitter feeds, or answer the question, “Did Kurtwood Smith have any major roles before he was in the original 1987 <em class="calibre7">RoboCop</em>?”<sup class="calibre20"><a id="calibre_link-1170" href="#calibre_link-34" class="calibre6">1</a></sup></p>
<p class="indentb">Since so much work on a computer involves going on the internet, it’d be great if your programs could get online. <em class="calibre7">Web scraping</em> is the term for using a program to download and process content from the web. For example, Google runs many web scraping programs to index web pages for <span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-824"></span>its search engine. In this chapter, you will learn about several modules that make it easy to scrape web pages in Python.</p>
<p class="hang"><span class="codestrong">webbrowser</span> Comes with Python and opens a browser to a specific page.</p>
<p class="hang"><span class="codestrong">requests</span> Downloads files and web pages from the internet.</p>
<p class="hang"><span class="codestrong">bs4</span> Parses HTML, the format that web pages are written in.</p>
<p class="hang"><span class="codestrong">selenium</span> Launches and controls a web browser. The <span class="literal">selenium</span> module is able to fill in forms and simulate mouse clicks in this browser.</p>
<h3 class="h1" id="calibre_link-373"><strong class="calibre2">Project: mapIt.py with the webbrowser Module</strong></h3>
<p class="noindent">The <span class="literal">webbrowser</span> module’s <span class="literal">open()</span> function can launch a new browser to a specified URL. Enter the following into the interactive shell:</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">import webbrowser</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">webbrowser.open('https://inventwithpython.com/')</span></p>
<p class="indent">A web browser tab will open to the URL <em class="calibre7"><a href="https://inventwithpython.com/" class="calibre6">https://inventwithpython.com/</a></em>. This is about the only thing the <span class="literal">webbrowser</span> module can do. Even so, the <span class="literal">open()</span> function does make some interesting things possible. For example, it’s tedious to copy a street address to the clipboard and bring up a map of it on Google Maps. You could take a few steps out of this task by writing a simple script to automatically launch the map in your browser using the contents of your clipboard. This way, you only have to copy the address to a clipboard and run the script, and the map will be loaded for you.</p>
<p class="indent">This is what your program does:</p>
<ol class="calibre10">
<li class="noindent1">Gets a street address from the command line arguments or clipboard</li>
<li class="noindent1">Opens the web browser to the Google Maps page for the address</li>
</ol>
<p class="indent">This means your code will need to do the following:</p>
<ol class="calibre10">
<li class="noindent1">Read the command line arguments from <span class="literal">sys.argv</span>.</li>
<li class="noindent1">Read the clipboard contents.</li>
<li class="noindent1">Call the <span class="literal">webbrowser.open()</span> function to open the web browser.</li>
</ol>
<p class="indent">Open a new file editor tab and save it as <em class="calibre7">mapIt.py</em>.</p>
<h4 class="h2" id="calibre_link-374"><strong class="calibre2"><em class="calibre9">Step 1: Figure Out the URL</em></strong></h4>
<p class="noindent">Based on the instructions in <a href="#calibre_link-35" class="calibre6">Appendix B</a>, set up <em class="calibre7">mapIt.py</em> so that when you run it from the command line, like so . . .</p>
<p class="programs">C:\&gt; mapit 870 Valencia St, San Francisco, CA 94110</p>
<p class="noindent">. . . the script will use the command line arguments instead of the clipboard. If there are no command line arguments, then the program will know to use the contents of the clipboard.</p>
<p class="indent"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-854"></span>First you need to figure out what URL to use for a given street address. When you load <em class="calibre7"><a href="https://maps.google.com/" class="calibre6">https://maps.google.com/</a></em> in the browser and search for an address, the URL in the address bar looks something like this: <em class="calibre7"><a href="https://www.google.com/maps/place/870+Valencia+St/@37.7590311,-122.4215096,17z/data=!3m1!4b1!4m2!3m1!1s0x808f7e3dadc07a37:0xc86b0b2bb93b73d8" class="calibre6">https://www.google.com/maps/place/870+Valencia+St/@37.7590311,-122.4215096,17z/data=!3m1!4b1!4m2!3m1!1s0x808f7e3dadc07a37:0xc86b0b2bb93b73d8</a></em>.</p>
<p class="indent">The address is in the URL, but there’s a lot of additional text there as well. Websites often add extra data to URLs to help track visitors or customize sites. But if you try just going to <em class="calibre7"><a href="https://www.google.com/maps/place/870+Valencia+St+San+Francisco+CA/" class="calibre6">https://www.google.com/maps/place/870+Valencia+St+San+Francisco+CA/</a></em>, you’ll find that it still brings up the correct page. So your program can be set to open a web browser to <span class="literal">'https://www.google.com/maps/place/</span><span class="codeitalic">your_address_string</span><span class="literal">'</span> (where <span class="codeitalic">your_address_string</span> is the address you want to map).</p>
<h4 class="h2" id="calibre_link-375"><strong class="calibre2"><em class="calibre9">Step 2: Handle the Command Line Arguments</em></strong></h4>
<p class="noindent">Make your code look like this:</p>
<p class="programs">#! python3<br class="calibre5" />
# mapIt.py - Launches a map in the browser using an address from the<br class="calibre5" />
# command line or clipboard.<br class="calibre5" /><br class="calibre5" />
import webbrowser, sys<br class="calibre5" />
if len(sys.argv) &gt; 1:<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;# Get address from command line.<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;address = ' '.join(sys.argv[1:])<br class="calibre5" /><br class="calibre5" />
# TODO: Get address from clipboard.</p>
<p class="indent">After the program’s <span class="literal">#!</span> shebang line, you need to import the <span class="literal">webbrowser</span> module for launching the browser and import the <span class="literal">sys</span> module for reading the potential command line arguments. The <span class="literal">sys.argv</span> variable stores a list of the program’s filename and command line arguments. If this list has more than just the filename in it, then <span class="literal">len(sys.argv)</span> evaluates to an integer greater than <span class="literal">1</span>, meaning that command line arguments have indeed been provided.</p>
<p class="indent">Command line arguments are usually separated by spaces, but in this case, you want to interpret all of the arguments as a single string. Since <span class="literal">sys.argv</span> is a list of strings, you can pass it to the <span class="literal">join()</span> method, which returns a single string value. You don’t want the program name in this string, so instead of <span class="literal">sys.argv</span>, you should pass <span class="literal">sys.argv[1:]</span> to chop off the first element of the array. The final string that this expression evaluates to is stored in the <span class="literal">address</span> variable.</p>
<p class="indent">If you run the program by entering this into the command line . . .</p>
<p class="programs">mapit 870 Valencia St, San Francisco, CA 94110</p>
<p class="noindent">. . . the <span class="literal">sys.argv</span> variable will contain this list value:</p>
<p class="programs">['mapIt.py', '870', 'Valencia', 'St, ', 'San', 'Francisco, ', 'CA', '94110']</p>
<p class="indent">The <span class="literal">address</span> variable will contain the string <span class="literal">'870 Valencia St, San Francisco, CA 94110'</span>.</p>
<h4 class="h2" id="calibre_link-376"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-1040" class="calibre1"></span><strong class="calibre2"><em class="calibre9">Step 3: Handle the Clipboard Content and Launch the Browser</em></strong></h4>
<p class="noindent">Make your code look like the following:</p>
<p class="programs">#! python3<br class="calibre5" />
# mapIt.py - Launches a map in the browser using an address from the<br class="calibre5" />
# command line or clipboard.<br class="calibre5" />
import webbrowser, sys<span class="codestrong1">,</span> <span class="codestrong1">pyperclip</span><br class="calibre5" />
if len(sys.argv) &gt; 1:<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;# Get address from command line.<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;address = ' '.join(sys.argv[1:])<br class="calibre5" />
<span class="codestrong1">else:</span><br class="calibre5" />
<span class="codestrong1">&nbsp;&nbsp;&nbsp;&nbsp;# Get address from clipboard.</span><br class="calibre5" />
<span class="codestrong1">&nbsp;&nbsp;&nbsp;&nbsp;address = pyperclip.paste()</span><br class="calibre5" /><br class="calibre5" />
<span class="codestrong1">webbrowser.open('https://www.google.com/maps/place/' + address)</span></p>
<p class="indent">If there are no command line arguments, the program will assume the address is stored on the clipboard. You can get the clipboard content with <span class="literal">pyperclip.paste()</span> and store it in a variable named <span class="literal">address</span>. Finally, to launch a web browser with the Google Maps URL, call <span class="literal">webbrowser.open()</span>.</p>

<h3 class="h1" id="calibre_link-378"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-917" class="calibre1"></span><strong class="calibre2">Downloading Files from the Web with the requests Module</strong></h3>
<p class="noindent">The <span class="literal">requests</span> module lets you easily download files from the web without having to worry about complicated issues such as network errors, connection problems, and data compression. The <span class="literal">requests</span> module doesn’t come with Python, so you’ll have to install it first. From the command line, run <span class="codestrong">pip install --user requests</span>. (<a href="#calibre_link-2" class="calibre6">Appendix A</a> has additional details on how to install third-party modules.)</p>
<p class="indent">The <span class="literal">requests</span> module was written because Python’s <span class="literal">urllib2</span> module is too complicated to use. In fact, take a permanent marker and black out this entire paragraph. Forget I ever mentioned <span class="literal">urllib2</span>. If you need to download things from the web, just use the <span class="literal">requests</span> module.</p>
<p class="indent">Next, do a simple test to make sure the <span class="literal">requests</span> module installed itself correctly. Enter the following into the interactive shell:</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">import requests</span></p>
<p class="indent">If no error messages show up, then the <span class="literal">requests</span> module has been successfully installed.</p>
<h4 class="h2" id="calibre_link-379"><strong class="calibre2"><em class="calibre9">Downloading a Web Page with the requests.get() Function</em></strong></h4>
<p class="noindent">The <span class="literal">requests.get()</span> function takes a string of a URL to download. By calling <span class="literal">type()</span> on <span class="literal">requests.get()</span>’s return value, you can see that it returns a <span class="literal">Response</span> object, which contains the response that the web server gave for your request. I’ll explain the <span class="literal">Response</span> object in more detail later, but for now, enter the following into the interactive shell while your computer is connected to the internet:</p>
<p class="programs">&nbsp;&nbsp;&nbsp;&gt;&gt;&gt; <span class="codestrong1">import requests</span><br class="calibre5" />
<span class="ent">➊</span> &gt;&gt;&gt; <span class="codestrong1">res = requests.get('https://automatetheboringstuff.com/files/rj.txt')</span><br class="calibre5" />
&nbsp;&nbsp;&nbsp;&gt;&gt;&gt; <span class="codestrong1">type(res)</span><br class="calibre5" />
&nbsp;&nbsp;&nbsp;&lt;class 'requests.models.Response'&gt;<br class="calibre5" />
<span class="ent">➋</span> &gt;&gt;&gt; <span class="codestrong1">res.status_code == requests.codes.ok</span><br class="calibre5" />
&nbsp;&nbsp;&nbsp;True<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&gt;&gt;&gt; <span class="codestrong1">len(res.text)</span><br class="calibre5" />
&nbsp;&nbsp;&nbsp;178981<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&gt;&gt;&gt; <span class="codestrong1">print(res.text[:250])</span><br class="calibre5" />
&nbsp;&nbsp;&nbsp;The Project Gutenberg EBook of Romeo and Juliet, by William Shakespeare<br class="calibre5" /><br class="calibre5" />
&nbsp;&nbsp;&nbsp;This eBook is for the use of anyone anywhere at no cost and with<br class="calibre5" />
&nbsp;&nbsp;&nbsp;almost no restrictions whatsoever.&nbsp;&nbsp;You may copy it, give it away or<br class="calibre5" />
&nbsp;&nbsp;&nbsp;re-use it under the terms of the Proje</p>
<p class="indent">The URL goes to a text web page for the entire play of <em class="calibre7">Romeo and Juliet</em>, provided on this book’s site <span class="ent">➊</span>. You can tell that the request for this web page succeeded by checking the <span class="literal">status_code</span> attribute of the <span class="literal">Response</span> object. If it is equal to the value of <span class="literal">requests.codes.ok</span>, then everything went fine <span class="ent">➋</span>. (Incidentally, the status code for “OK” in the HTTP protocol is 200. You may already be familiar with the 404 status code for “Not Found.”) <span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-1073"></span>You can find a complete list of HTTP status codes and their meanings at <em class="calibre7"><a href="https://en.wikipedia.org/wiki/List_of_HTTP_status_codes" class="calibre6">https://en.wikipedia.org/wiki/List_of_HTTP_status_codes</a></em>.</p>
<p class="indent">If the request succeeded, the downloaded web page is stored as a string in the <span class="literal">Response</span> object’s <span class="literal">text</span> variable. This variable holds a large string of the entire play; the call to <span class="literal">len(res.text)</span> shows you that it is more than 178,000 characters long. Finally, calling <span class="literal">print(res.text[:250])</span> displays only the first 250 characters.</p>
<p class="indent">If the request failed and displayed an error message, like “Failed to establish a new connection” or “Max retries exceeded,” then check your internet connection. Connecting to servers can be quite complicated, and I can’t give a full list of possible problems here. You can find common causes of your error by doing a web search of the error message in quotes.</p>
<h4 class="h2" id="calibre_link-380"><strong class="calibre2"><em class="calibre9">Checking for Errors</em></strong></h4>
<p class="noindent">As you’ve seen, the <span class="literal">Response</span> object has a <span class="literal">status_code</span> attribute that can be checked against <span class="literal">requests.codes.ok</span> (a variable that has the integer value <span class="literal">200</span>) to see whether the download succeeded. A simpler way to check for success is to call the <span class="literal">raise_for_status()</span> method on the <span class="literal">Response</span> object. This will raise an exception if there was an error downloading the file and will do nothing if the download succeeded. Enter the following into the interactive shell:</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">res = requests.get('https://inventwithpython.com/page_that_does_not_exist')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">res.raise_for_status()</span><br class="calibre5" />
Traceback (most recent call last):<br class="calibre5" />
&nbsp;&nbsp;File "&lt;stdin&gt;", line 1, in &lt;module&gt;<br class="calibre5" /><br class="calibre5" />
&nbsp;&nbsp;File "C:\Users\Al\AppData\Local\Programs\Python\Python37\lib\site-packages\requests\models<br class="calibre5" />
.py", line 940, in raise_for_status<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;raise HTTPError(http_error_msg, response=self)<br class="calibre5" />
requests.exceptions.HTTPError: 404 Client Error: Not Found for url: https://inventwithpython<br class="calibre5" />
.com/page_that_does_not_exist.html</p>
<p class="indent">The <span class="literal">raise_for_status()</span> method is a good way to ensure that a program halts if a bad download occurs. This is a good thing: You want your program to stop as soon as some unexpected error happens. If a failed download <em class="calibre7">isn’t</em> a deal breaker for your program, you can wrap the <span class="literal">raise_for_status()</span> line with <span class="literal">try</span> and <span class="literal">except</span> statements to handle this error case without crashing.</p>
<p class="programs">import requests<br class="calibre5" />
res = requests.get('https://inventwithpython.com/page_that_does_not_exist')<br class="calibre5" />
try:<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;res.raise_for_status()<br class="calibre5" />
except Exception as exc:<br class="calibre5" />
&nbsp;&nbsp;&nbsp;&nbsp;print('There was a problem: %s' % (exc))</p>
<p class="indent">This <span class="literal">raise_for_status()</span> method call causes the program to output the following:</p>
<p class="programs">There was a problem: 404 Client Error: Not Found for url: https://<br class="calibre5" />
inventwithpython.com/page_that_does_not_exist.html</p>
<p class="indent"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-916"></span>Always call <span class="literal">raise_for_status()</span> after calling <span class="literal">requests.get()</span>. You want to be sure that the download has actually worked before your program continues.</p>
<h3 class="h1" id="calibre_link-381"><strong class="calibre2">Saving Downloaded Files to the Hard Drive</strong></h3>
<p class="noindent">From here, you can save the web page to a file on your hard drive with the standard <span class="literal">open()</span> function and <span class="literal">write()</span> method. There are some slight differences, though. First, you must open the file in <em class="calibre7">write binary</em> mode by passing the string <span class="literal">'wb'</span> as the second argument to <span class="literal">open()</span>. Even if the page is in plaintext (such as the <em class="calibre7">Romeo and Juliet</em> text you downloaded earlier), you need to write binary data instead of text data in order to maintain the <em class="calibre7">Unicode encoding</em> of the text.</p>
<p class="indent">To write the web page to a file, you can use a <span class="literal">for</span> loop with the <span class="literal">Response</span> object’s <span class="literal">iter_content()</span> method.</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">import requests</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">res = requests.get('https://automatetheboringstuff.com/files/rj.txt')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">res.raise_for_status()</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">playFile = open('RomeoAndJuliet.txt', 'wb')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">for chunk in res.iter_content(100000):</span><br class="calibre5" />
<span class="codestrong1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;playFile.write(chunk)</span><br class="calibre5" /><br class="calibre5" />
100000<br class="calibre5" />
78981<br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">playFile.close()</span></p>
<p class="indent">The <span class="literal">iter_content()</span> method returns “chunks” of the content on each iteration through the loop. Each chunk is of the <em class="calibre7">bytes</em> data type, and you get to specify how many bytes each chunk will contain. One hundred thousand bytes is generally a good size, so pass <span class="literal">100000</span> as the argument to <span class="literal">iter_content()</span>.</p>
<p class="indent">The file <em class="calibre7">RomeoAndJuliet.txt</em> will now exist in the current working directory. Note that while the filename on the website was <em class="calibre7">rj.txt</em>, the file on your hard drive has a different filename. The <span class="literal">requests</span> module simply handles downloading the contents of web pages. Once the page is downloaded, it is simply data in your program. Even if you were to lose your internet <span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-1003"></span>connection after downloading the web page, all the page data would still be on your computer.</p>
<div class="sidebar">
<p class="sidebart"><strong class="calibre4">UNICODE ENCODINGS</strong></p>
<p class="spara">Unicode encodings are beyond the scope of this book, but you can learn more about them from these web pages:</p>
<ul class="calibre8">
<li class="noindent1">Joel on Software: The Absolute Minimum Every Software Developer Absolutely, Positively Must Know About Unicode and Character Sets (No Excuses!): <em class="calibre7"><a href="https://www.joelonsoftware.com/articles/Unicode.html" class="calibre6">https://www.joelonsoftware.com/articles/Unicode.html</a></em></li>
<li class="noindent1">Pragmatic Unicode: <em class="calibre7"><a href="https://nedbatchelder.com/text/unipain.html" class="calibre6">https://nedbatchelder.com/text/unipain.html</a></em></li>
</ul>
</div>
<p class="indent">The <span class="literal">write()</span> method returns the number of bytes written to the file. In the previous example, there were 100,000 bytes in the first chunk, and the remaining part of the file needed only 78,981 bytes.</p>
<p class="indent">To review, here’s the complete process for downloading and saving a file:</p>
<ol class="calibre10">
<li class="noindent1">Call <span class="literal">requests.get()</span> to download the file.</li>
<li class="noindent1">Call <span class="literal">open()</span> with <span class="literal">'wb'</span> to create a new file in write binary mode.</li>
<li class="noindent1">Loop over the <span class="literal">Response</span> object’s <span class="literal">iter_content()</span> method.</li>
<li class="noindent1">Call <span class="literal">write()</span> on each iteration to write the content to the file.</li>
<li class="noindent1">Call <span class="literal">close()</span> to close the file.</li>
</ol>
<p class="indent">That’s all there is to the <span class="literal">requests</span> module! The <span class="literal">for</span> loop and <span class="literal">iter_content()</span> stuff may seem complicated compared to the <span class="literal">open()</span>/<span class="literal">write()</span>/<span class="literal">close()</span> workflow you’ve been using to write text files, but it’s to ensure that the <span class="literal">requests</span> module doesn’t eat up too much memory even if you download massive files. You can learn about the <span class="literal">requests</span> module’s other features from <em class="calibre7"><a href="https://requests.readthedocs.org/" class="calibre6">https://requests.readthedocs.org/</a></em>.</p>

<h4 class="h2" id="calibre_link-384"><strong class="calibre2"><em class="calibre9">A Quick Refresher</em></strong></h4>
<p class="noindent">In case it’s been a while since you’ve looked at any HTML, here’s a quick overview of the basics. An HTML file is a plaintext file with the <em class="calibre7">.html</em> file extension. The text in these files is surrounded by <em class="calibre7">tags</em>, which are words enclosed in angle brackets. The tags tell the browser how to format the web page. A starting tag and closing tag can enclose some text to form an <em class="calibre7">element</em>. The <em class="calibre7">text</em> (or <em class="calibre7">inner HTML</em>) is the content between the starting and closing tags. For example, the following HTML will display <em class="calibre7">Hello, world!</em> in the browser, with <em class="calibre7">Hello</em> in bold:</p>
<p class="programs">&lt;strong&gt;Hello&lt;/strong&gt;, world!</p>

<p class="indent">The opening <span class="literal">&lt;strong&gt;</span> tag says that the enclosed text will appear in bold. The closing <span class="literal">&lt;/strong&gt;</span> tags tells the browser where the end of the bold text is.</p>
<p class="indent">There are many different tags in HTML. Some of these tags have extra properties in the form of <em class="calibre7">attributes</em> within the angle brackets. For example, the <span class="literal">&lt;a&gt;</span> tag encloses text that should be a link. The URL that the text links to is determined by the <span class="literal">href</span> attribute. <div class="sidebar">
<p class="sidebart"><strong class="calibre4">DON’T USE REGULAR EXPRESSIONS TO PARSE HTML</strong></p>
<p class="spara">Locating a specific piece of HTML in a string seems like a perfect case for regular expressions. However, I advise you against it. There are many different ways that HTML can be formatted and still be considered valid HTML, but trying to capture all these possible variations in a regular expression can be tedious and error prone. A module developed specifically for parsing HTML, such as <span class="literal1">bs4</span>, will be less likely to result in bugs.</p>
<p class="sparai">You can find an extended argument for why you shouldn’t parse HTML with regular expressions at <em class="calibre7"><a href="https://stackoverflow.com/a/1732454/1893164/" class="calibre6">https://stackoverflow.com/a/1732454/1893164/</a></em>.</p>
</div>

<p class="noindent">Once your program has downloaded a web page using the <span class="literal">requests</span> module, you will have the page’s HTML content as a single string value. <p class="indent">This is where the browser’s developer tools can help. Before writing any code, do a little research. If you visit the site and search for the 94105 ZIP code, the site will take you to a page showing the forecast for that area.</p>
<p class="indent">What if you’re interested in scraping the weather information for that ZIP code? Right-click where it is on the page (or <small class="calibre11">CONTROL</small>-click on macOS) and select <strong class="calibre4">Inspect Element</strong> from the context menu that appears. This will bring up the Developer Tools window, which shows you the HTML that produces this particular part of the web page. <a href="#calibre_link-41" class="calibre6">Figure 12-5</a> shows the developer tools open to the HTML of the nearest forecast. Note that if the <em class="calibre7"><a href="https://weather.gov/" class="calibre6">https://weather.gov/</a></em> site changes the design of its web pages, you’ll need to repeat this process to inspect the new elements.</p>

<h3 class="h1" id="calibre_link-388"><strong class="calibre2">Parsing HTML with the bs4 Module</strong></h3>
<p class="noindent">Beautiful Soup is a module for extracting information from an HTML page (and is much better for this purpose than regular expressions). The Beautiful Soup module’s name is <span class="literal">bs4</span> (for Beautiful Soup, version 4). To install it, you will need to run <span class="literal">pip install --user beautifulsoup4</span> from the command line. (Check out <a href="#calibre_link-2" class="calibre6">Appendix A</a> for instructions on installing third-party modules.) While <span class="literal">beautifulsoup4</span> is the name used for installation, to import Beautiful Soup you run <span class="literal">import bs4</span>.</p>
<p class="indent">For this chapter, the Beautiful Soup examples will <em class="calibre7">parse</em> (that is, analyze and identify the parts of) an HTML file on the hard drive. Open a new file editor tab in Mu, enter the following, and save it as <em class="calibre7">example.html</em>. Alternatively, download it from <em class="calibre7"><a href="https://nostarch.com/automatestuff2/" class="calibre6">https://nostarch.com/automatestuff2/</a></em>.</p>
<p class="programs">&lt;!-- This is the example.html example file. --&gt;<br class="calibre5" /><br class="calibre5" />
&lt;html&gt;&lt;head&gt;&lt;title&gt;The Website Title&lt;/title&gt;&lt;/head&gt;<br class="calibre5" />
&lt;body&gt;<br class="calibre5" />
&lt;p&gt;Download my &lt;strong&gt;Python&lt;/strong&gt; book from &lt;a href="https://<br class="calibre5" />
inventwithpython.com"&gt;my website&lt;/a&gt;.&lt;/p&gt;<br class="calibre5" />
&lt;p class="slogan"&gt;Learn Python the easy way!&lt;/p&gt;<br class="calibre5" />
&lt;p&gt;By &lt;span id="author"&gt;Al Sweigart&lt;/span&gt;&lt;/p&gt;<br class="calibre5" />
&lt;/body&gt;&lt;/html&gt;</p>
<p class="indent">As you can see, even a simple HTML file involves many different tags and attributes, and matters quickly get confusing with complex websites. Thankfully, Beautiful Soup makes working with HTML much easier.</p>
<h4 class="h2" id="calibre_link-389"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-825" class="calibre1"></span><strong class="calibre2"><em class="calibre9">Creating a BeautifulSoup Object from HTML</em></strong></h4>
<p class="noindent">The <span class="literal">bs4.BeautifulSoup()</span> function needs to be called with a string containing the HTML it will parse. The <span class="literal">bs4.BeautifulSoup()</span> function returns a <span class="literal">BeautifulSoup</span> object. Enter the following into the interactive shell while your computer is connected to the internet:</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">import requests, bs4</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">res = requests.get('https://nostarch.com')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">res.raise_for_status()</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">noStarchSoup = bs4.BeautifulSoup(res.text, 'html.parser')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">type(noStarchSoup)</span><br class="calibre5" />
&lt;class 'bs4.BeautifulSoup'&gt;</p>
<p class="indent">This code uses <span class="literal">requests.get()</span> to download the main page from the No Starch Press website and then passes the <span class="literal">text</span> attribute of the response to <span class="literal">bs4.BeautifulSoup()</span>. The <span class="literal">BeautifulSoup</span> object that it returns is stored in a variable named <span class="literal">noStarchSoup</span>.</p>
<p class="indent">You can also load an HTML file from your hard drive by passing a <span class="literal">File</span> object to <span class="literal">bs4.BeautifulSoup()</span> along with a second argument that tells Beautiful Soup which parser to use to analyze the HTML.</p>
<p class="indent">Enter the following into the interactive shell (after making sure the <em class="calibre7">example.html</em> file is in the working directory):</p>
<p class="programs">&gt;&gt;&gt; <span class="codestrong1">exampleFile = open('example.html')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">exampleSoup = bs4.BeautifulSoup(exampleFile, 'html.parser')</span><br class="calibre5" />
&gt;&gt;&gt; <span class="codestrong1">type(exampleSoup)</span><br class="calibre5" />
&lt;class 'bs4.BeautifulSoup'&gt;</p>
<p class="indent">The <span class="literal">'html.parser'</span> parser used here comes with Python. However, you can use the faster <span class="literal">'lxml'</span> parser if you install the third-party <span class="literal">lxml</span> module. Follow the instructions in <a href="#calibre_link-2" class="calibre6">Appendix A</a> to install this module by running <span class="literal">pip install --user lxml</span>. <p class="indent">Once you have a <span class="literal">BeautifulSoup</span> object, you can use its methods to locate specific parts of an HTML document.</p>

<h3 class="h1" id="calibre_link-403"><strong class="calibre2">Controlling the Browser with the selenium Module</strong></h3>
<p class="noindent">The <span class="literal">selenium</span> module lets Python directly control the browser by programmatically clicking links and filling in login information, almost as though there were a human user interacting with the page. Using <span class="literal">selenium</span>, you can interact with web pages in a much more advanced way than with <span class="literal">requests</span> and <span class="literal">bs4</span>; but because it launches a web browser, it is a bit slower and hard to run in the background if, say, you just need to download some files from the web.</p>
<p class="indent">Still, if you need to interact with a web page in a way that, say, depends on the JavaScript code that updates the page, you’ll need to use <span class="literal">selenium</span> instead of <span class="literal">requests</span>. That’s because major ecommerce websites such as Amazon almost certainly have software systems to recognize traffic that they suspect is a script harvesting their info or signing up for multiple free accounts. These sites may refuse to serve pages to you after a while, breaking any scripts you’ve made. The <span class="literal">selenium</span> module is much more likely to function on these sites long-term than <span class="literal">requests</span>.</p>
<p class="indent">A major “tell” to websites that you’re using a script is the <em class="calibre7">user-agent</em> string, which identifies the web browser and is included in all HTTP requests. For example, the user-agent string for the <span class="literal">requests</span> module is something like <span class="literal">'python-requests/2.21.0'</span>. You can visit a site such as <em class="calibre7"><a href="https://www.whatsmyua.info/" class="calibre6">https://www.whatsmyua.info/</a></em> to see your user-agent string. Using <span class="literal">selenium</span>, you’re much more likely to “pass for human” because not only is Selenium’s user-agent is the same as a regular browser (for instance, <span class="literal">'Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:65.0) Gecko/20100101 Firefox/65.0'</span>), but it has the same traffic patterns: a <span class="literal">selenium</span>-controlled browser will download images, advertisements, cookies, and privacy-invading trackers just like a regular browser. However, <span class="literal">selenium</span> can still be detected by websites, and major ticketing and ecommerce websites often block browsers controlled by <span class="literal">selenium</span> to prevent web scraping of their pages.</p>

<h4 class="h2" id="calibre_link-406"><span {http://www.idpf.org/2007/ops}type="pagebreak" id="calibre_link-845" class="calibre1"></span><strong class="calibre2"><em class="calibre9">Clicking the Page</em></strong></h4>

<h3 class="h1" id="calibre_link-411"><strong class="calibre2">Summary</strong></h3>
<p class="noindent">Most boring tasks aren’t limited to the files on your computer. Being able to programmatically download web pages will extend your programs to the internet. The <span class="literal">selenium</span> module will allow you to log in to websites and fill out forms automatically. Since a web browser is the most common way to send and receive information over the internet, this is a great ability to have in your programmer toolkit.</p>
